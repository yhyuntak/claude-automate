# 지식 검색 및 활용

> 축적된 지식을 작업 중 자연스럽게 활용

---

## User Story

개발자가 비슷한 문제를 만났을 때, 과거에 축적된 인사이트가 자동으로 제안되어 더 빠르게 해결할 수 있다.

## Acceptance Criteria

- [ ] 키워드 기반 검색 기능
- [ ] 컨텍스트 기반 자동 추천 (작업 중 관련 지식 제안)
- [ ] 지식 활용 피드백 (유용했는지 기록)
- [ ] 검색 결과 랭킹 (최신성, 관련성, 유용성)

## 비기능 요구사항

- 속도: 1초 이내 검색 결과
- 관련성: 노이즈 최소화
- 학습: 피드백 기반 개선

## Dependencies

- phase2-001 완료 (PARA 구조)
- phase2-002 완료 (축적된 지식 필요)

## 구현 노트 (작업 중 추가)

### 검색 방식

1. **명시적 검색**: `/knowledge search {query}`
2. **암시적 추천**: 작업 중 관련 지식 자동 표시

### 매칭 전략

- 키워드 매칭 (기본)
- 파일 경로 매칭 (같은 영역 작업 시)
- 에러 메시지 매칭 (비슷한 에러 경험)

### 활용 시나리오

```
상황: TypeScript 제네릭 에러 발생
시스템: "관련 인사이트 발견: 'TypeScript 제네릭 추론 문제 해결'"
개발자: 클릭하여 확인 → 문제 해결
시스템: "이 인사이트가 도움이 되었나요?" → 피드백 기록
```
