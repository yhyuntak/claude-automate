# Triage 워크플로우 구현

> 리뷰 결과를 우선순위화하여 의사결정 지원

---

## User Story

여러 리뷰 에이전트의 결과가 나오면, Triage 시스템이 중요도/긴급도에 따라 정렬하여 개발자가 효율적으로 검토할 수 있게 한다.

## Acceptance Criteria

- [ ] 리뷰 결과 분류 기준 정의 (Critical, High, Medium, Low)
- [ ] 자동 분류 로직 구현
- [ ] 통합 리포트 포맷 정의
- [ ] 개발자가 빠르게 판단할 수 있는 요약 제공

## 비기능 요구사항

- UX: 5초 내에 전체 상황 파악 가능
- 정확도: Critical 이슈 누락 없음

## Dependencies

- phase1-003 완료 후 시작 (병렬 리뷰 에이전트 필요)
