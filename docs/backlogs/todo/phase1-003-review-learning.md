# 리뷰 결과 학습 축적

> 리뷰에서 발견된 패턴을 시스템에 축적

---

## User Story

리뷰에서 반복적으로 발견되는 이슈는 프로젝트 규칙으로 축적되어, 다음 리뷰에서 더 빠르게 감지되고 개발자도 미리 인지할 수 있다.

## Acceptance Criteria

- [ ] 리뷰 결과에서 패턴 추출 로직
- [ ] 추출된 패턴을 rules/에 저장하는 포맷
- [ ] pattern-checker가 새 규칙을 인식
- [ ] 축적된 규칙 조회 기능

## 비기능 요구사항

- 자동화: 수동 개입 없이 축적
- 품질: 노이즈 필터링 (일회성 이슈 제외)

## Dependencies

- phase1-001 완료 후 시작 가능
- phase1-002 결과 활용 가능

---

## 구현 노트 (작업 중 추가)

### 핵심 아이디어

> "배운 것을 시스템에 축적 → 복리 효과"

한 번 발견한 이슈는 다시 수동으로 검토하지 않음

### 축적 대상

1. **프로젝트 특화 패턴**: 이 프로젝트에서만 적용되는 규칙
2. **팀 컨벤션**: 팀이 결정한 스타일/구조
3. **반복 실수**: 같은 개발자가 반복하는 패턴

### 축적 포맷 (초안)

```markdown
# Rule: {rule-id}

> {한 줄 설명}

## 패턴

{감지할 코드 패턴}

## 권장 사항

{어떻게 수정해야 하는지}

## 근거

{왜 이것이 문제인지}

## 출처

- 발견 세션: {session-id}
- 발견 횟수: {count}
```
