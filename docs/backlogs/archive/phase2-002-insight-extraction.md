# 세션 인사이트 자동 추출

> 세션에서 발생한 배움을 자동으로 포착하고 분류

---

## User Story

개발자가 세션 중 문제를 해결하거나 새로운 패턴을 발견했을 때, 그 인사이트가 자동으로 추출되어 지식 베이스에 축적된다.

## Acceptance Criteria

- [ ] 인사이트 감지 트리거 정의 (어떤 상황에서 추출?)
- [ ] 추출 에이전트 구현 (Haiku tier)
- [ ] 분류 로직 (PARA 카테고리 자동 판단)
- [ ] context-builder와 연동

## 비기능 요구사항

- 비침습적: 작업 흐름 방해하지 않음
- 정확성: 노이즈 최소화, 의미 있는 것만
- 편집 가능: 자동 추출 후 수동 수정 허용

## Dependencies

- phase1-002 완료 후 시작 (PARA 구조 필요)

---

## 구현 노트 (작업 중 추가)

### 인사이트 유형

1. **문제-해결 패턴**: "이 에러는 이렇게 해결했다"
2. **아키텍처 결정**: "이 구조를 선택한 이유"
3. **안티패턴 발견**: "이렇게 하면 안 된다"
4. **도구 팁**: "이 도구는 이렇게 쓰면 효과적"

### 감지 트리거 (초안)

- `/wrap` 호출 시 세션 분석
- "결국", "알고 보니", "해결책은" 등 키워드
- 에러 → 성공 전환 패턴

### 추출 포맷

```markdown
# Insight: {title}

**유형**: {problem-solution | architecture | antipattern | tip}
**발견일**: YYYY-MM-DD
**세션**: {session-id}

## 컨텍스트
{어떤 상황에서 발견했는지}

## 핵심 내용
{인사이트 본문}

## 관련 파일
- {file1}
- {file2}
```
