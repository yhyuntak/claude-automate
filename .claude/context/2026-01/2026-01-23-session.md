## Decisions Made

1. **English as default documentation language**
   - Reason: 20-30% token reduction vs Korean; broader international audience
   - Impact: Reduces API costs for all future agent interactions

2. **Skills-based rule distribution over plugin rules**
   - Reason: Rules/ directory doesn't auto-load; Skills have contextual auto-activation
   - Impact: interaction-rules skill provides consistent UX patterns system-wide
   - Trade-off: Requires skill to be activated; automatic via context detection

3. **Comprehensive README expansion**
   - Reason: Serves as single source of truth for new users and agents
   - Impact: Reduces need for agents to search multiple files for core concepts
   - Trade-off: README now 1000+ lines; manageable via clear section structure

4. **Hierarchical documentation structure**
   - agents/, commands/, skills/ directories mirror plugin execution model
   - Reason: Clear mental model for what each component does
   - Impact: Easier onboarding for new developers extending the plugin
