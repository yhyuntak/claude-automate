## 핵심 아키텍처 결정

### 레이어 매핑 (토스테크 모델 기반)

```
Commands    =  Controller       (진입점, 사용자 인터페이스)
Agents      =  Service Layer    (비즈니스 로직, 분석/검증)
Skills      =  Domain Component (단일 책임, 재사용 가능)
MCP         =  Infrastructure   (외부 연동, 어댑터)
CLAUDE.md   =  package.json     (프로젝트 정체성)
rules/*.md  =  eslint.config    (세부 규칙)
```

**진화 원칙**: 이 모델은 시작점. 더 좋은 아이디어가 생기면 재정의 가능.

### 3-Tier 모델 선택

| Tier | 모델 | 용도 | 비용 |
|------|------|------|------|
| Low | Haiku | 데이터 수집, 단순 패턴 매칭 | $ |
| Medium | Sonnet | 분석, 의사결정, 표준 작업 | $$ |
| High | Opus | 복잡한 충돌 해결, 전략적 결정 | $$$ |

### Progressive Disclosure

```
CLAUDE.md (항상 로드 - 원칙만)
    ↓ 필요시
rules/*.md (세부 규칙)
    ↓ 필요시
docs/references/*.md (참고 자료)
    ↓ 필요시
skills/*/references/*.md (스킬 상세)
```

## 영감의 원천

### Peter Steinberger (Moltbot)

### Kieran Klaassen (Code Review)
> "13개 전문 AI 리뷰어. 50/50 규칙: 리뷰 50%, 시스템 개선 50%."

- Triage 기반 의사결정
- 병렬 실행으로 시간 단축
- 학습 결과 축적

### 토스테크 (Software 3.0)
> "코드를 보면 안 된다. 상위 레벨 개념을 확고히."

- 레이어드 아키텍처 유지
- 안티패턴은 그대로 적용
